# github_iva_zetovic_vjezba_jedan
Upoznavanje s osnovama gita
1. Što je visual Studio Code i za što se najčešće koristi?
To je lagan, besplatan editor koda koji se koristi za pisanje, uređivanje i debugiranje koda.
2. Kako se zove panel u kojem vidiš sve datoteke i mape svog projekta?
Explorer.
3. Kako možeš instalirati ekstenzije u VS Codeu i čemu one služe?
U panelu Extensions( ikonica kvadratića ) ---> "Install". Služe za dodavanje funkcionalnosti(jezici, alati, teme).
4. Kako otvoriti postojeći projekt ili mapu u VS Code-u?
File---> Open Folder i odabereš mapu projekta.
5. Što se sve može raditi u Exploreru?
Pregledavati, otvarati, kreirati, brisati, premještati datoteke i mape.
6. Kako možeš brzo pronaći određenu datoteku u projektu?
Pritiskom na Ctrl + P i upisivanjem imena datoteke.
7. Kako možeš istovremeno gledati dvije različite datoteke u editoru?
Desni klik ----> Split Editor ili ikonica za podjelu ekrana.
8. Kako pokrećeš automatsko formatiranje koda u VS Code-u?
Shift + Alt + F. 
9. Što je version control i zašto se koristi?
Sustav za praćenje promjena nad kodom. Koristi se zbog sigurnosti, suradnje i  vraćanja na stare verzije.
10. Kako sse u VS Code-u otvara panel za Git(Source Control)?
Klik na ikonicu Source Control( tri grananja)u lijevom panelu.
11. Što znače commit, push, pull?
commit- spremanje promjena lokalno; push- slanje promjena na GitHub; pull- preuzimanje novih promjena s GitHuba
12. Čemu služi commit poruka i kako treba izgledati? 
Objašnjava što si promijenio. Treba biti kratka i jasna(npr. "Popravljen bug u loginu.").
13. Kako možeš vidjeti koje si promjene napravio u pojedinoj datoteci?
U Source Control panelu klikneš na datoteku ---> prikazuje diff.
14. Kreiraš novu datoteku. Koji su koraci da se promjena pošalje na GitHub?
 prvi korak git add . drugi korak git commit-m "poruka" treći korak git push
 15. Kako se vraćaš na stariju verziju datoteke ako si pogriješio? 
 Pregledaš stare commitove---> Checkout, Restore ili git checkout <commit>.
 16. Na koji način možeš provjeriti status projekta i koje datoteke su izmijenjene?
 U Source Control panelu ili naredbom git status.
